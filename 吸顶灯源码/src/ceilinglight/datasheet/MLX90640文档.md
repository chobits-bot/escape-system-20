# MLX90640 热成像传感器数据获取与温度计算指南（火灾监测应用）

本文档基于 MLX90640 数据手册（Rev 12），面向 ESP32（IDF 5.2.6）开发，旨在实现局部空间火灾监测的热成像系统。

------

## 1. 硬件连接与 I2C 通信

### 1.1 引脚定义





| 引脚 | 名称 | 功能               |
| :--- | :--- | :----------------- |
| 1    | SDA  | I2C 数据线（双向） |
| 2    | VDD  | 电源 +3.3V         |
| 3    | GND  | 地                 |
| 4    | SCL  | I2C 时钟线（输入） |

### 1.2 电气参数

- 供电电压：3.3V ± 0.05V（推荐）
- 工作电流：<23mA
- I2C 时钟：最高 1MHz（建议 400kHz）
- 默认从机地址：**0x33**（7-bit）

### 1.3 ESP32 接线

text

```
ESP32 3.3V  → MLX90640 VDD
ESP32 GND   → MLX90640 GND
ESP32 GPIO21 → MLX90640 SDA (上拉电阻 2.2k~4.7kΩ)
ESP32 GPIO22 → MLX90640 SCL (上拉电阻 2.2k~4.7kΩ)
```



- 在 ESP32 上使用 I2C 外设，配置为主机模式，频率 400kHz。

## 2. 数据读取流程

### 2.1 初始化与配置

- 上电后等待至少 40ms（POR 时间）。
- 读取 EEPROM 中的校准参数（详见第 4 节）。
- 配置 Control Register 1（地址 0x800D）：
  - 刷新率（例如 4Hz，即 `011`）
  - 分辨率（建议 18-bit，即 `10`）
  - 读取模式：**Chess 模式**（默认，性能最佳）
- 建议将配置写入 EEPROM（地址 0x240C），以免每次上电重新配置。

### 2.2 获取状态与数据就绪

- 读取 Status Register（0x8000）的 bit 3（`New data available in RAM`）。
- 当该位为 1 时，表示新的一帧（子页）已更新。
- 读取完成后，通过写 1 清零该位（或写入 0 清除，参考手册）。

### 2.3 读取 RAM 数据

- RAM 基地址：0x0400 ~ 0x07FF（部分区域）。
- 需要读取的内容：
  - 768 个像素的原始 IR 值（每个 16-bit，有符号，补码）。
  - 补偿像素（CP）数据：子页0 和 子页1（地址 0x0708 和 0x0728）。
  - PTAT 电压（地址 0x0720）。
  - VDD 电压（地址 0x072A）。
  - VBE 电压（地址 0x0700）。
  - 增益 RAM 值（地址 0x070A）。
- **重要**：MLX90640 使用双子页交替更新，必须读取两个子页的完整数据才能得到完整一帧（每个子页 384 个像素，共 768）。读取模式（Chess 或 TV）决定了像素在 RAM 中的排列。

------

## 3. 数据排列与子页模式

### 3.1 Chess 模式（推荐）

- 子页0 和 子页1 交叉排列，每个子页包含棋盘格中的一半像素。
- 合并方式：按行列奇偶组合。
  - 像素 `(row, col)` 属于子页0 还是子页1，由公式决定（详见数据手册 11.2.2.7 节）。
- 合并后得到完整的 32x24 矩阵。

### 3.2 读取建议

- 每次检测到新数据位，读取当前子页并存储。
- 当两个子页均完成读取，合并成一帧。

## 4. 校准参数恢复（从 EEPROM）

计算温度需要从 EEPROM（地址 0x2400~0x273F）读取大量校准系数。以下列出关键参数及其恢复方法（以像素(12,16)为例，但大部分是公共的）。

### 4.1 公共参数





| 参数                  | EEPROM 地址    | 解码方法                                                     |
| :-------------------- | :------------- | :----------------------------------------------------------- |
| **VDD 校正**          | 0x2433         | 提取高8位作为 `K_Vdd`，低8位作为 `Vdd_25`，均为补码，公式见手册 11.1.1。 |
| **PTAT 校正**         | 0x2432         | 提取 `Kv_PTAT`、`Kt_PTAT`，并读取 `PTAT_25`（0x2431）。      |
| **GAIN**              | 0x2430         | 16位有符号。                                                 |
| **Kta_avg**           | 0x2436         | 按行列奇偶分组，有 4 个平均值（行奇/偶 × 列奇/偶）。         |
| **Kv_avg**            | 0x2434         | 类似分组。                                                   |
| **KsTa**              | 0x243C 的高8位 | 公式 `Ksta = Ksta_EE / 2^13`。                               |
| **TGC**               | 0x243C 的低8位 | 对于 `BAA/BAB` 型号，TGC 固定为 0，不可改。                  |
| **KsTo 斜率**         | 0x243D~0x243E  | 对应四个温度范围的斜率。                                     |
| **转角温度 CT3, CT4** | 0x243F         | 解码得到台阶温度，CT1=-40°C, CT2=0°C。                       |
| **分辨率校准**        | 0x2438 高2位   | `Resolution_EE`，用于校正当前 ADC 分辨率。                   |

### 4.2 像素专属参数（每个像素 3 个 16-bit 字）

- 地址范围 0x2440 ~ 0x273F。
- 每个像素占用 3 个字，分别存储：
  - 字1：符号位 + 偏移量（6 bit）+ 异常标志（bit0）。
  - 字2：灵敏度 α（10 bit）+ 异常标志。
  - 字3：Kta 系数（3 bit）+ 异常标志。
- 解码方法：
  - **偏移量**：从字1提取高6位（有符号），然后结合行/列校正（`OCC_row/col`）和公共偏移平均值（`Pix_os_average`）计算出 `pix_os_ref`，再经 `Kta` 和 `Kv` 补偿（详见 11.1.3）。
  - **灵敏度 α**：从字2提取 10 位，除以 `2^(Alpha_scale)`，其中 `Alpha_scale` 从 0x2420 高4位解码得到。
  - **Kta 像素**：从字3提取 3 位，结合公共 Kta 平均值和缩放系数。

> **建议**：在初始化时一次性读取所有 EEPROM 校准数据并缓存，避免每次读取时重复读取。

## 5. 温度计算步骤

### 5.1 计算供电电压 Vdd

text

```
Vdd = (Resolution_corr * RAM[0x072A] - Vdd_25) / K_Vdd + 3.3
```



- `Resolution_corr = 2^(Resolution_EE) / 2^(Resolution_REG)`，其中 `Resolution_REG` 从控制寄存器读取。

### 5.2 计算环境温度 Ta

text

```
Ta = ( (Vptat_art / (1 + Kv_PTAT * ΔV)) - Vptat_25 ) / Kt_PTAT + 25
```



- `Vptat_art = (Vptat * 2^18) / (Vptat * Alpha_PTAT + Vbe)`
- `Vptat = RAM[0x0720]`（补码转有符号），`Vbe = RAM[0x0700]`，`Alpha_PTAT` 从 0x2410 高4位计算。
- `ΔV = (RAM[0x072A] - Vdd_25) / K_Vdd`

### 5.3 像素数据补偿（每个像素）

#### 5.3.1 增益补偿

text

```
pix_gain = RAM[pixel_data] * GAIN / RAM[0x070A]
```



（RAM[0x070A] 为增益原始值，也需补码转换）

#### 5.3.2 偏移补偿（考虑 Ta 和 Vdd）

text

```
pix_os = pix_gain - offset_ref * (1 + Kta_pix * (Ta - 25)) * (1 + Kv_pix * (Vdd - 3.3))
```



- `offset_ref` 由 `pix_os_average` + 行列校正得到（参见手册 11.1.3）。
- `Kta_pix` 和 `Kv_pix` 根据像素行列奇偶从公共值 + 像素专属 Kta 计算得到。

#### 5.3.3 发射率补偿（默认 ε=1）

text

```
V_IR_comp = pix_os / ε
```



### 5.4 补偿像素（CP）处理

- 分别对两个子页的 CP 数据（地址 0x0708 和 0x0728）做增益补偿和偏移补偿（使用 CP 专用的偏移、Kta、Kv 系数）。
- **建议**：对 CP 数据进行移动平均滤波（如窗口 16），以减少噪声对梯度补偿的影响。

### 5.5 梯度补偿（使用 CP 数据）

- `pix_os_corrected = V_IR_comp - TGC * (CP_subpage - CP_ambient)`，其中 TGC 通常为 0（BAA/BAB 型号），可忽略。

### 5.6 灵敏度归一化

text

```
alpha_comp = alpha_pix * (1 + KsTa * (Ta - 25)) * Alpha_corr_range
```



- `Alpha_corr_range` 根据目标温度范围（见下节）选择。

### 5.7 计算目标温度 To

使用四阶 Steffensen 迭代公式（简化版）：

text

```
Ta_K4 = (Ta + 273.15)^4
Tr_K4 = (Tr + 273.15)^4   // Tr 为反射温度，默认取 Ta - 8°C
T_a_r = Tr_K4 - (Tr_K4 - Ta_K4) / ε   // 此处 ε=1，所以 T_a_r = Ta_K4

Sx = KsTo_x * (alpha_comp^3 * V_IR_comp + alpha_comp^4 * T_a_r)^(1/4)
To = [ V_IR_comp / ( alpha_comp * (1 + KsTo_x * 273.15) ) + Sx ]^(1/4) - 273.15
```



- `KsTo_x` 和 `Alpha_corr_range` 根据当前计算的 `To` 所在温度范围选择（范围1: -40~0°C, 范围2: 0~CT3, 范围3: CT3~CT4, 范围4: >CT4）。
- **第一次计算**：先假设在范围2（0~CT3），用 `KsTo2` 计算得到一个初步 To。
- 然后根据初步 To 实际落入哪个范围，重新选择参数，重新计算最终 To（迭代一次通常足够）。

## 6. 生成热成像图

### 6.1 数据归一化

- 得到所有 768 个像素的温度值（单位 °C）。
- 找出帧内最高温度 `T_max` 和最低温度 `T_min`。

### 6.2 伪彩色映射（用于显示）

- 常用映射：将温度线性映射到色彩（如彩虹色或热度图）。
- 对于火灾监测，建议突出高温区域（如 >60°C 显示红色预警）。

### 6.3 图像分辨率

- 物理像素 32×24，可上采样至更大尺寸（如插值）以便查看。

### 6.4 火灾监测策略

- 设定绝对温度阈值（如 80°C）和温差阈值（如局部与背景温差 > 30°C）。
- 实时监测最高温度及热点区域，触发报警。
- 刷新率选择：推荐 4~8Hz，平衡响应速度和噪声。
- 噪声抑制：对每个像素可做时间域滤波（如一阶低通或移动平均），减少误报。

## 7. ESP32（IDF 5.2.6）实现要点

### 7.1 I2C 驱动

- 使用 `i2c_master_init()` 配置，频率 400kHz。
- 使用 `i2c_master_transmit_receive()` 或 `i2c_master_cmd_begin()` 发送读写命令。

### 7.2 读取流程（伪代码）

c

```
// 1. 初始化并配置
mlx90640_init();

// 2. 循环
while(1) {
    // 检查状态寄存器 bit3
    uint16_t status = read_register(0x8000);
    if (status & (1<<3)) {
        // 读取当前子页数据
        uint16_t subpage = (read_register(0x800D) >> 4) & 0x01; // 0 or 1
        read_subpage(subpage); // 读取对应子页的 384 个像素及 CP 等
        
        // 清除新数据标志
        write_register(0x8000, status | (1<<3)); // 写1清除
        
        // 累计两个子页，若都收到则合并并计算温度
        if (subpage == 1) {
            merge_frames();
            calculate_temperatures();
            generate_thermal_image();
            check_fire_alarm();
        }
    }
    vTaskDelay(pdMS_TO_TICKS(10));
}
```



### 7.3 注意事项

- EEPROM 读取一次后缓存到内存，避免重复 I2C 操作。
- 浮点运算较多，可使用 `float` 或定点数优化（ESP32 有 FPU）。
- 考虑使用 FreeRTOS 任务管理刷新和报警逻辑。

------

## 8. 针对火灾监测的优化建议

- **快速响应**：选择较高刷新率（如 8Hz），但噪声会增大，可配合滤波。
- **动态阈值**：结合环境温度 Ta 自动调整报警阈值（例如：热点 > Ta + 50°C）。
- **区域监测**：可将图像分为若干区域，分别检测区域平均温度或最高温度。
- **镜头清洁**：确保传感器窗口无灰尘或水汽，否则影响准确性。
- **热稳定**：设备上电后需等待约 4 分钟达到热平衡，初期数据不可靠。
