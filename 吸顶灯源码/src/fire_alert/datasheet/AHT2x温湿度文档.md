# AHT20 温湿度传感器 I2C 通信与数据读取指南

> 本指南基于 AHT20 数据手册（V1.0，2019/12）整理，适用于通过标准 I2C 总线读取温湿度数据的开发人员。

------

## 1. 硬件连接要点





| 引脚 | 名称 | 说明                           |
| :--- | :--- | :----------------------------- |
| 1    | NC   | 悬空                           |
| 2    | VDD  | 电源电压 2.0~5.5V（推荐 3.3V） |
| 3    | SCL  | I2C 时钟线（需上拉）           |
| 4    | SDA  | I2C 数据线（需上拉）           |
| 5    | GND  | 电源地                         |
| 6    | NC   | 悬空                           |

**关键电路要求**：

- VDD 与 GND 之间必须接 **4.7µF** 去耦电容，尽量靠近传感器。
- SCL 和 SDA 需外接上拉电阻（典型值 **10kΩ**）。
- MCU 供电电压必须与传感器 VDD 一致。
- **I2C 总线上只能连接单个 AHT20**，不能挂载其他 I2C 器件。
- 如需更高可靠性，可在 VDD 上增加 RC 滤波（参考典型电路）。

------

## 2. I2C 通信基础

- **设备地址**：`0x38`（7 位地址，不含读写位）。
- 支持标准模式（100 kHz）和快速模式（400 kHz）。
- 所有命令、数据均为 **MSB 在前**。

------

## 3. 完整读取流程（分步详解）

### 3.1 上电与初始化（仅需一次）

1. **上电**后，等待至少 **40 ms**（传感器内部初始化）。
2. **检查校准使能位**（Bit[3]）：
   - 发送命令 `0x71`（读取 1 字节状态字）。
   - 状态字 Bit[3] 若为 `1`，表示已校准，可跳过初始化。
   - 若为 `0`，则需发送**初始化命令**（软复位可替代）。

**初始化命令格式**：

- I2C 起始条件 → 发送设备地址 + 写位（`0x38 << 1 | 0`）→ 发送命令字节 `0xBE` → 发送参数字节 `0x08` → 发送参数字节 `0x00` → 停止条件。

> 初始化仅需在首次上电或异常复位后执行，正常连续采集时无需重复。

------

### 3.2 触发温湿度测量

发送 **触发测量命令**（`0xAC`）：





| 步骤 | I2C 操作          | 数据                     |
| :--- | :---------------- | :----------------------- |
| 1    | 起始条件          | -                        |
| 2    | 发送设备地址 + 写 | `0x38 << 1`（即 `0x70`） |
| 3    | 等待 ACK          | -                        |
| 4    | 发送命令字节      | `0xAC`                   |
| 5    | 等待 ACK          | -                        |
| 6    | 发送参数字节 1    | `0x33`                   |
| 7    | 等待 ACK          | -                        |
| 8    | 发送参数字节 2    | `0x00`                   |
| 9    | 等待 ACK          | -                        |
| 10   | 停止条件          | -                        |

**注意**：发送触发命令后，传感器进入测量状态，需要等待 **至少 75 ms** 才能读取结果。

------

### 3.3 读取温湿度数据（6 字节）

1. 等待 ≥75 ms 后，发送 **读取命令**（`0x71`）：
   - 起始条件 → 设备地址 + 写（`0x70`）→ 发送 `0x71` → 停止条件。
2. 重新发起 **读操作**：
   - 起始条件 → 设备地址 + 读（`0x38 << 1 | 1`，即 `0x71`）→ 连续读取 **6 个字节**（每字节后主机应答 ACK，最后一字节后发送 NACK）→ 停止条件。

**读取到的 6 字节结构**：





| 字节   | 内容                                                         |
| :----- | :----------------------------------------------------------- |
| Byte 0 | 状态字（含忙标志 Bit[7] 和校准标志 Bit[3]）                  |
| Byte 1 | 湿度数据高 8 位（S<sub>RH</sub>[19:12]）                     |
| Byte 2 | 湿度数据中 8 位（S<sub>RH</sub>[11:4]）                      |
| Byte 3 | 湿度数据低 4 位 + 温度数据高 4 位（S<sub>RH</sub>[3:0] 与 S<sub>T</sub>[19:16]） |
| Byte 4 | 温度数据中 8 位（S<sub>T</sub>[15:8]）                       |
| Byte 5 | 温度数据低 8 位（S<sub>T</sub>[7:0]）                        |

**状态字解读**（Byte 0）：

- Bit[7]：`1` 表示传感器忙（测量未完成），`0` 表示数据有效。
- Bit[3]：`1` 表示已校准，`0` 表示需要初始化。

**如果 Bit[7] 为 1**，需重新等待并再次读取，直到该位变为 0。

------

### 3.4 数据拼接与物理量转换

#### 拼接 20 位原始值

- **湿度原始值 S<sub>RH</sub>**（20 位）：

  text

  ```
  S_RH = (Byte1 << 12) | (Byte2 << 4) | (Byte3 >> 4)
  ```

  

- **温度原始值 S<sub>T</sub>**（20 位）：

  text

  ```
  S_T = ((Byte3 & 0x0F) << 16) | (Byte4 << 8) | Byte5
  ```

  

#### 转换为物理值

- **相对湿度**（%RH）：

  RH=SRH220×100%*R**H*=220*S**R**H*×100%

- **温度**（℃）：

  T=ST220×200−50*T*=220*S**T*×200−50

**示例**：
若 S<sub>RH</sub> = 0xCCCCD（约 838861），则 RH ≈ 80.0%
若 S<sub>T</sub> = 0x19999A（约 1677722），则 T ≈ 30.0℃

## 4. 软复位（可选）

发送命令 `0xBA`（参数 `0x00`, `0x00`）可软复位传感器，效果等同于重新初始化，复位时间 ≤20 ms。

------

## 5. 关键注意事项

### 5.1 时序与延时

- 上电后必须等待 ≥40 ms 才能操作。
- 触发测量后必须等待 ≥75 ms 再读取，否则状态位 Bit[7] 会保持 1，数据无效。

### 5.2 测量频率与自热

- 为保证自身温升 <0.1℃，**激活时间不应超过测量周期的 10%**。建议 **每 2 秒测量 1 次**（即测量间隔 ≥2s）。

### 5.3 焊接后处理

- 回流焊后，需将传感器置于 **>75% RH** 环境中至少 **24 小时**（或自然环境 >40% RH 下 5 天以上）进行聚合物**重新水合**，否则读数会漂移。使用低温回流焊（如 180℃）可缩短水合时间。
- **焊接后禁止冲洗电路板**，应使用免洗焊锡膏。

### 5.4 存储与环境

- 长期暴露于 >80% RH 或化学蒸汽中会导致暂时性漂移，可通过“恢复处理”恢复：
  - **烘干**：80~85℃，<5% RH，保持 10 小时；
  - **重新水合**：20~30℃，>75% RH，保持 24 小时。
- 避免接触挥发性胶水、胶带、泡沫等材料，生产区域应通风良好。

### 5.5 PCB 布局建议

- 传感器应远离发热元件，可在 PCB 上开**铣削狭缝**以隔离热传导。
- 确保传感器通风口与外界空气充分接触，减少测量盲区。

### 5.6 I2C 信号完整性

- SCL 和 SDA 不应长距离平行走线，建议在两者之间布置 GND 或 VDD 隔离，或使用屏蔽电缆。
- 必要时降低 SCL 频率可提高抗干扰能力。

## 6. 典型 I2C 读取代码伪代码（C 语言风格）

c

```
// 假设 I2C 底层函数：i2c_start(), i2c_write_byte(), i2c_read_byte(ack), i2c_stop()

#define AHT20_ADDR         0x38
#define CMD_INIT           0xBE
#define CMD_TRIGGER        0xAC
#define CMD_SOFT_RESET     0xBA
#define CMD_READ_STATUS    0x71

// 初始化（上电后调用一次）
void aht20_init(void) {
    i2c_start();
    i2c_write_byte((AHT20_ADDR << 1) | 0); // 写
    i2c_write_byte(CMD_INIT);
    i2c_write_byte(0x08);
    i2c_write_byte(0x00);
    i2c_stop();
    delay_ms(20); // 等待初始化完成
}

// 触发测量并读取数据
int aht20_read(float *temp, float *humi) {
    uint8_t data[6];
    
    // 1. 触发测量
    i2c_start();
    i2c_write_byte((AHT20_ADDR << 1) | 0);
    i2c_write_byte(CMD_TRIGGER);
    i2c_write_byte(0x33);
    i2c_write_byte(0x00);
    i2c_stop();
    
    // 2. 等待测量完成
    delay_ms(75);
    
    // 3. 读取 6 字节数据
    i2c_start();
    i2c_write_byte((AHT20_ADDR << 1) | 1); // 读
    for (int i = 0; i < 6; i++) {
        data[i] = i2c_read_byte(i < 5); // 前5字节ACK，最后一字节NACK
    }
    i2c_stop();
    
    // 4. 检查状态位
    if (data[0] & 0x80) { // Bit[7] == 1 表示忙
        return -1; // 数据无效
    }
    if (!(data[0] & 0x08)) { // Bit[3] == 0 表示未校准
        aht20_init();
        return -2;
    }
    
    // 5. 拼合原始值
    uint32_t s_rh = ((uint32_t)data[1] << 12) | ((uint32_t)data[2] << 4) | (data[3] >> 4);
    uint32_t s_t  = ((uint32_t)(data[3] & 0x0F) << 16) | ((uint32_t)data[4] << 8) | data[5];
    
    // 6. 转换物理量
    *humi = (s_rh / 1048576.0) * 100.0;   // 2^20 = 1048576
    *temp = (s_t / 1048576.0) * 200.0 - 50.0;
    
    return 0;
}
```

## 7. 常见问题排查





| 现象                       | 可能原因                 | 解决办法                      |
| :------------------------- | :----------------------- | :---------------------------- |
| 读取状态字 Bit[7] 一直为 1 | 等待时间不足             | 确保触发后延迟 ≥75 ms         |
| 湿度/温度值严重偏差        | 焊接后未重新水合         | 按 2.3 节进行恢复处理         |
| I2C 通信无 ACK             | 地址错误或上拉电阻缺失   | 检查地址为 0x38，检查上拉电阻 |
| 读数随测量频率升高而偏移   | 传感器自热               | 降低测量频率（≤0.5Hz）        |
| 长期使用后漂移             | 暴露于化学蒸汽或极端湿度 | 执行恢复处理（烘干+水合）     |
