# ENS160 空气质量传感器 I2C 通信指南

本文档基于 ScioSense ENS160 数据手册（版本 1.3），提供通过 I2C 总线正确读取传感器数据的完整操作流程。

------

## 1. 硬件连接





| ENS160 引脚       | 功能         | 连接至 MCU                         |
| :---------------- | :----------- | :--------------------------------- |
| VDD (Pin 4)       | 主电源       | 1.71V ~ 1.98V                      |
| VDDIO (Pin 5)     | 接口电源     | 1.71V ~ 3.6V（与MCU逻辑电平匹配）  |
| VSS (Pin 8/9)     | 地           | GND                                |
| SDA (Pin 1)       | I2C 数据线   | MCU SDA（需上拉电阻）              |
| SCL (Pin 2)       | I2C 时钟线   | MCU SCL（需上拉电阻）              |
| CSn (Pin 7)       | 接口选择     | 拉高至 VDDIO（选择 I2C）           |
| MISO/ADDR (Pin 3) | I2C 地址选择 | 拉低 → 地址 0x52；拉高 → 地址 0x53 |
| INTn (Pin 6)      | 中断输出     | （可选）连接 MCU 中断引脚          |

- **上拉电阻**：SCL 和 SDA 需外接上拉电阻，阻值取决于总线速率（推荐 2.2kΩ ~ 10kΩ）。
- **去耦电容**：在 VDD 和 VDDIO 引脚附近放置 0.1µF 电容。

------

## 2. I2C 地址





| ADDR 引脚电平  | 7-bit 地址 | 写地址 (0x) | 读地址 (0x) |
| :------------- | :--------- | :---------- | :---------- |
| 低电平 (GND)   | 0x52       | 0xA4        | 0xA5        |
| 高电平 (VDDIO) | 0x53       | 0xA6        | 0xA7        |

*注意：地址在芯片上电时锁存，更改需重新上电。*

------

## 3. I2C 时序参数（快速模式，400kHz）





| 参数           | 最小值 | 单位    |
| :------------- | :----- | :------ |
| SCL 频率       | -      | 400 kHz |
| START 保持时间 | 0.6    | µs      |
| SCL 低电平周期 | 1.3    | µs      |
| SCL 高电平周期 | 0.6    | µs      |
| 数据建立时间   | 100    | ns      |
| 数据保持时间   | 0      | ns      |
| STOP 建立时间  | 0.6    | µs      |

**支持标准 (100k)、快速 (400k) 和快速+ (1M) 模式。**

## 4. 初始化流程

### 4.1 上电后读取 PART_ID（验证通信）

- **寄存器**：`0x00`（2 字节，小端）
- **期望值**：`0x0160`（LSB 0x60, MSB 0x01）
- **操作**：读取 2 个字节并校验。

### 4.2 设置操作模式 (OPMODE)

- **寄存器**：`0x10`
- **可用值**：
  - `0x00`：深度睡眠（低功耗）
  - `0x01`：空闲模式（配置用）
  - `0x02`：标准气体传感模式（**正常工作**）
  - `0xF0`：复位（会重置芯片）

**建议流程**：

1. 上电后，写入 `0x01` 进入空闲模式。
2. 可选：配置 CONFIG 寄存器（见下节）。
3. 写入 `0x02` 启动气体传感。

### 4.3 配置中断 (CONFIG)（可选）

- **寄存器**：`0x11`
- **位定义**：
  - BIT6 (INTPOL)：中断极性（0=低有效，1=高有效）
  - BIT5 (INT_CFG)：驱动类型（0=开漏，1=推挽）
  - BIT1 (INTDAT)：新 DATA_XXX 数据到达时触发中断
  - BIT0 (INTEN)：总中断使能
- **典型配置**：`0x23`（低有效，开漏，新数据触发中断）

### 4.4 写入温湿度补偿数据（可选但推荐）

- **TEMP_IN**：`0x13`（2 字节，小端）
  - 格式：温度 (K) × 64，例如 25°C → (25+273.15)*64 = 19081.6 → 取整 19082 (0x4A8A)
  - **注意**：先写低字节 (0x13)，再写高字节 (0x14)，高字节写入时触发更新。
- **RH_IN**：`0x15`（2 字节，小端）
  - 格式：相对湿度 (%) × 512，例如 50% → 50*512 = 25600 (0x6400)
  - 同样先写低字节再写高字节。

## 5. 读取空气质量数据

### 5.1 检查状态 (DEVICE_STATUS)

- **寄存器**：`0x20`（1 字节）
- **位域**：
  - BIT7 (STATAS)：=1 表示操作模式正在运行
  - BIT6 (STATER)：=1 表示错误
  - BIT[3:2] (VALIDITY FLAG)：
    - `00`：正常
    - `01`：暖机阶段（Warm-up，上电后前3分钟）
    - `10`：初始启动阶段（Initial Start-up，首次使用前1小时）
    - `11`：无效输出（传感器异常）
  - BIT1 (NEWDAT)：=1 表示 DATA_XXX 有新的有效数据（读取任一 DATA 寄存器后自动清零）
  - BIT0 (NEWGPR)：=1 表示 GPR_READ 有新数据

**建议**：每次读取数据前检查 `NEWDAT` 位，若为 1 则读取；同时检查 `VALIDITY FLAG` 确保数据有效。

### 5.2 读取主要空气质量参数

所有 DATA_XXX 寄存器均为只读，支持连续读取（自动地址递增）。





| 参数        | 寄存器地址 | 大小 (字节) | 单位     | 范围           |
| :---------- | :--------- | :---------- | :------- | :------------- |
| AQI         | 0x21       | 1           | UBA 指数 | 1 ~ 5          |
| TVOC        | 0x22       | 2 (小端)    | ppb      | 0 ~ 65,000     |
| eCO2        | 0x24       | 2 (小端)    | ppm      | 400 ~ 65,000   |
| 温度 (内部) | 0x30       | 2 (小端)    | °C       | 格式同 TEMP_IN |
| 湿度 (内部) | 0x32       | 2 (小端)    | %RH      | 格式同 RH_IN   |

**读取示例**（连续读取从 0x21 到 0x25 共 5 个字节）：

- 发送写地址 + 0x21
- 发送读地址，读取 5 字节
  - Byte0 = AQI
  - Byte1-2 = TVOC (LSB first)
  - Byte3-4 = eCO2 (LSB first)

**注意**：`DATA_TVOC` 和 `DATA_ETH` 地址均为 0x22，默认输出 TVOC（乙醇校准版本）。

## 6. 完整读取例程（伪代码）

c

```
// 1. 初始化 I2C 总线

// 2. 校验设备 ID
uint16_t part_id = i2c_read_16(0x00); // 小端
if (part_id != 0x0160) error();

// 3. 设置空闲模式
i2c_write_byte(0x10, 0x01);
delay_ms(10);

// 4. （可选）配置中断，例如 0x23
i2c_write_byte(0x11, 0x23);

// 5. （可选）写入温湿度补偿
uint16_t temp_k_x64 = (uint16_t)((temp_celsius + 273.15) * 64);
i2c_write_16_le(0x13, temp_k_x64); // 先低后高
uint16_t rh_x512 = (uint16_t)(rh_percent * 512);
i2c_write_16_le(0x15, rh_x512);

// 6. 启动标准模式
i2c_write_byte(0x10, 0x02);

// 7. 循环读取数据
while (1) {
    uint8_t status = i2c_read_byte(0x20);
    if (status & 0x40) { // STATER 错误位
        // 处理错误
    }
    uint8_t validity = (status >> 2) & 0x03;
    if (validity != 0) {
        // 数据尚不可靠 (暖机/启动中)，可等待或忽略
    }
    if (status & 0x02) { // NEWDAT
        // 读取 AQI, TVOC, eCO2 (从0x21开始连续读5字节)
        uint8_t buf[5];
        i2c_read_bytes(0x21, buf, 5);
        uint8_t aqi = buf[0];
        uint16_t tvoc = buf[1] | (buf[2] << 8);
        uint16_t eco2 = buf[3] | (buf[4] << 8);
        // 处理数据
    }
    delay_ms(1000); // 轮询间隔，或使用中断
}
```

## 7. 其他有用命令（通过 COMMAND 寄存器 0x12）

- **获取固件版本**：写入 `0x0E` 到 0x12，然后从 GPR_READ (0x48~0x4F) 读取版本信息（参见数据手册 Table 21）。
- **清除 GPR_READ**：写入 `0xCC`。

*注意：COMMAND 命令仅在 IDLE 模式 (0x01) 下执行有效。*

## 8. 数据有效性注意事项





| 阶段           | 持续时间          | 数据可靠性                     |
| :------------- | :---------------- | :----------------------------- |
| 初始启动       | 首次上电后 1 小时 | 算法正在学习基线，数据逐渐稳定 |
| 暖机 (Warm-up) | 每次上电后 3 分钟 | 数据可用但可能波动             |
| 正常工作       | 之后              | 数据稳定可靠                   |

建议在初始启动 1 小时内，不要将数据用于控制决策；暖机 3 分钟内可忽略或标记数据。

## 9. 常见问题排查

- **读取数据全为 0**：检查 OPMODE 是否为 0x02；检查 I2C 通信是否正常（PART_ID 验证）。
- **数据保持不变**：确认 NEWDAT 位是否置位，若一直为 0 则传感器可能未启动或处于错误状态。
- **地址错误**：确认 ADDR 引脚电平，上电后不可更改。
- **错误状态位 (STATER=1)**：可能是写入无效命令或 OPMODE 值非法，重新写入正确值。

## 11. 注意事项

- **电源去耦**：VDD 和 VDDIO 必须独立且靠近芯片放置去耦电容。
- **CSn** 必须始终拉高（接 VDDIO）以保持 I2C 模式。
- **不使用时**可写入 OPMODE 0x00 进入深度睡眠以降低功耗（约 10µA）。
- 传感器首次使用需经历 1 小时初始启动，期间基线自学习，数据会逐渐改善。
