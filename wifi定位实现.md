### **RSSI + 指纹库 + 图论路径规划**（混合定位）

#### 核心原理

1. 离线阶段（提前准备）：
   - 在楼层内划分网格（比如每 2m×2m 一个参考点）。
   - 用手机在每个参考点采集 3~4 个 ESP32 的 RSSI 值，建立**指纹数据库**（位置 → [RSSI1, RSSI2, RSSI3, RSSI4]）。
   - 把指纹数据存入 Redis 或 InfluxDB。
2. 在线定位阶段（实时）：
   - 用户手机连接任意一个 ESP32（或只是扫描 WiFi，不一定连接）。
   - 手机把当前扫描到的 3~4 个 RSSI 值上传到服务器。
   - 服务器用 **KNN / 余弦相似度 / 神经网络** 匹配最接近的指纹点，得到用户大致坐标（精度通常 3~7 米）。
3. 逃生指示：
   - 把用户位置映射到你们已有的**楼层拓扑图**（graph.sql 中的节点）。
   - 使用**蚁群算法**（或 A*、Dijkstra）计算**当前危险系数最低的逃生路径**。
   - 返回方向指引（“前方 8 米左转上楼梯” 或箭头 + 距离）。

#### 优点

- 3~4 个设备就能达到较好覆盖。
- 不需要额外硬件（手机 WiFi 就够）。
- 能融合传感器实时数据（温度、烟感、火焰）调整路径危险权重。
- 火灾时 WiFi 信号虽然会受影响，但仍能工作（比蓝牙、UWB 更稳健）。

### 具体实现建议

**1. ESP32 端（固件修改）**

- 让每个 ESP32 作为 **WiFi AP + Station** 双模式，或统一用一个主 AP + 其他 Mesh。
- 每 1~2 秒广播一次自己的 **设备 ID + 位置标签**。
- 支持手机扫描或连接后上报 RSSI（ESP32 可以读取 RSSI）。

**2. 手机端**

- App 或小程序：
  - 扫描 WiFi（WifiManager 或 startScan）。
  - 每 2~3 秒上传一次 {timestamp, ssid/mac列表, rssi列表}。
- 可选：让用户手动连接某个 ESP32 的 WiFi（更稳定）。

**3. 后端处理流程**

```
手机 → 上报 RSSI → escape.go 
    ↓
从 Redis/InfluxDB 取指纹库 + 当前传感器数据
    ↓
Python 算法服务：
   - 定位匹配（KNN 或简单加权质心）
   - 计算各路径危险系数（温度、火焰、烟感）
   - 蚁群/A* 规划最优逃生路径（支持跨楼层，禁用电梯）
    ↓
返回：用户坐标 + 逃生方向 + 语音/文字提示
```

**4. 简单起步版（快速验证）** 如果指纹库工作量太大，可以先用**加权质心法**（不需要大量采集）：

- 把每个 ESP32 的坐标固定。
- 根据 RSSI 强度计算加权平均位置。
- 公式示例：位置 = Σ (坐标_i * weight_i) / Σ weight_i（weight 与 RSSI 负相关）。
