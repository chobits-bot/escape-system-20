# 火灾逃生系统 - 3D建筑模型说明

## 项目概述

基于 Three.js 的 3D 建筑逃生模拟系统，集成蚁群算法（ACO）路径规划、动态逃生演示、火焰/烟雾特效等功能。支持 6 层建筑模型，4 个楼梯、4 个出口，可交互式模拟火灾逃生场景。

---

## 模型结构

### 建筑规格

| 参数 | 值 |
|------|-----|
| 楼层数 | 6 层（1F-6F） |
| 层高 | 4.5 米 |
| 建筑总高度 | 27 米 |
| 楼梯数量 | 4 个（东/西/北/南） |
| 出口数量 | 4 个（东/西/南/北） |
| 电梯数量 | 1 个（火灾模式禁用） |

### 每层布局

| 节点类型 | 数量 | 说明 |
|---------|------|------|
| 房间（room） | 3 个 | 办公室/会议室/机房等 |
| 走廊（corridor） | 3 个 | 主走廊 + 东西走廊 |
| 楼梯（stair） | 4 个 | 东/西/北/南楼梯 |
| 电梯（elevator） | 1 个 | 火灾时禁用 |

### 楼层功能分布

| 楼层 | 功能 |
|------|------|
| 1F | 大厅 + 商铺（4间） |
| 2F | 办公室 + 会议室 |
| 3F | 展厅 + 机房 + 休息区 |
| 4F | 办公室 + 培训室 |
| 5F | 实验室 + 图书馆 + 会议室 |
| 6F | 机房 + 休息区 + 大厅 |

## 3D 模型样式

### 建筑外观

- **楼板**：深灰色平面（`#2a2f3a`）
- **墙壁**：白色半透明（`#d8dee9`），带边线
- **门**：蓝色半开状态（`#88c0d0`）
- **窗户**：浅蓝色玻璃（`#7dd3fc`）
- **外墙**：灰色半透明（`#94a3b8`），带窗户开洞
- **屋顶**：深灰色（`#475569`）

### 室内设施

| 设施 | 样式 | 位置 |
|------|------|------|
| 楼梯 | 黄色台阶 + 扶手 | 4 个楼梯间 |
| 电梯 | 紫色半透明箱体 + 双开门 | 每层1个 |
| 消防栓 | 红色圆柱 + 接口管 | 1F |
| 监控室 | 蓝色地板 + 玻璃墙 + 显示屏 | 1F |
| 配电间 | 黄色警示地板 + 电箱 | 1F |

### 家具布局

根据房间类型自动布置：

| 房间类型 | 家具 |
|---------|------|
| 商铺 | 展示柜 + 收银台 |
| 办公室 | L型桌 + 显示器 + 文件柜 + 椅子 |
| 会议室 | 椭圆桌 + 围椅 |
| 机房 | 服务器机柜排列 |
| 休息区 | L型沙发 + 茶几 |

### 吸顶灯

- **数量**：每层 6 个（2x3 网格分布）
- **样式**：圆形薄盘（`#fff7ed` 暖白色）
- **光源**：PointLight 向下发光
- **火灾时**：起火点附近变为橙红色警告

## 功能模块

### 1. 3D 视图

- **鼠标操作**：左键旋转、右键平移、滚轮缩放
- **键盘操作**：WASD/方向键移动相机
- **视角预设**：透视、俯视、正视、侧视、适配
- **相机面板**：实时显示和调整相机位置/目标

### 2. 2D 平面图

- **楼层切换**：1F-6F 标签页切换
- **节点显示**：不同颜色区分节点类型
- **路径叠加**：显示计算出的逃生路径
- **图例**：房间、走廊、出口、楼梯、电梯等

### 3. 逃生路径计算

- **算法选择**：蚁群算法（ACO）— 其他算法预留
- **起点选择**：下拉框选择或点击 3D 模型放置
- **火灾模式**：开启后电梯禁用，路径避开电梯
- **起火点选择**：选择节点作为火灾起源

### 4. 逃生模拟

- **小人模型**：蓝色衣服 + 黄色头部，放大 2 倍
- **行走动画**：腿部/手臂摆动
- **移动方式**：沿 ACO 计算路径平滑移动
- **速度控制**：0.5x ~ 3x 可调

### 5. 火焰/烟雾效果

- **火焰粒子**：橙红色，向上跳动 + 闪烁
- **烟雾粒子**：灰色，缓慢上升扩散
- **火焰光源**：橙红色 PointLight 闪烁
- **影响范围**：起火点及相邻节点

### 6. 楼层高亮

- **模拟时**：小人所在楼层高亮，其他楼层半透明
- **跨楼层**：小人移动时自动切换高亮楼层
- **模拟结束**：所有楼层恢复正常显示

## 使用方法

### 启动服务

```bash
# 1. 启动后端容器
cd ~/escape-project
docker compose up -d

# 2. 启动算法服务
cd ~/escape-project/aiot/algorithm-service
nohup python3 main.py > /tmp/algo.log 2>&1 &

# 3. 用 tmux 启动前端（断线不丢失）
tmux new -d -s frontend "cd ~/escape-project/aiot/aiot-vue && npm run dev -- --host 0.0.0.0 --port 5173"
```

### 访问系统

浏览器打开 `http://服务器IP:5173`，登录后进入 **楼层模型** 页面。

### 逃生模拟流程

1. **选择起点**：从下拉框选择，或点击"📍 点击放置小人"在 3D 模型上点击放置
2. **设置起火点**（可选）：在模拟工具栏选择起火节点
3. **开启火灾模式**（可选）：电梯禁用，路径避开电梯
4. **计算路径**：点击"🔥 计算逃生路径"
5. **开始模拟**：点击"▶ 开始"
6. **观察效果**：
   - 小人沿路径行走
   - 路径上指示灯变绿色箭头
   - 起火点冒火焰 + 烟雾
   - 起火点附近吸顶灯变橙红色
   - 楼层自动切换高亮

### 相机操作

| 操作 | 说明 |
|------|------|
| 左键拖拽 | 旋转视角 |
| 右键拖拽 | 平移 |
| 滚轮 | 缩放 |
| WASD / 方向键 | 移动相机 |
| Q / 空格 | 上升 |
| E / Shift | 下降 |

### 语言切换

右上角 "中文 / English" 切换界面语言。

## 常见问题

### Q: 前端显示"获取数据失败"
A: 检查后端容器是否运行：`docker ps`，确认 MySQL 密码正确。

### Q: 计算逃生路径失败
A: 确认算法服务运行：`curl http://localhost:8090/health`，返回 `{"status":"ok"}`。

### Q: SSH 断开后前端停止
A: 使用 tmux 启动：`tmux new -d -s frontend "npm run dev -- --host 0.0.0.0 --port 5173"`

### Q: 服务器重启后服务丢失
A: 需要重新启动 Docker 容器和算法服务，或配置 systemd 自动启动。
