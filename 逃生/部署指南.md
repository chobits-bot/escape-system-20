# 逃生系统云服务器部署指南

## 一、前置条件

1. 一台云服务器（阿里云/腾讯云，2核4G 起步）
2. 已安装 Docker + Docker Compose
3. 云服务器安全组已开放端口

## 二、安全组端口配置

| 端口 | 协议 | 用途 | 是否必须 |
|------|------|------|----------|
| 22 | TCP | SSH 远程管理 | 是 |
| 1883 | TCP | EMQX MQTT（设备通信） | 是 |
| 8080 | TCP | Go 后端 API | 是 |
| 5173 | TCP | Vue 前端页面 | 是 |
| 8086 | TCP | InfluxDB（可内网访问） | 可选 |
| 18083 | TCP | EMQX Dashboard | 可选 |

## 三、服务器部署

### 3.1 上传项目到服务器

```bash
# 在本地打包项目
scp -r C:\Users\choby\Desktop\逃生 root@你的公网IP:/opt/aiot
```

或用 Git：
```bash
cd /opt
git clone <你的仓库地址> aiot
```

### 3.2 进入项目目录

```bash
cd /opt/aiot
```

### 3.3 创建必要目录

```bash
mkdir -p sql
# 复制数据库初始化脚本
cp aiot/aiot-v2/manifest/deploy/sql/aiot.sql sql/
cp aiot/aiot-v2/manifest/deploy/sql/graph.sql sql/
```

### 3.4 启动所有服务

```bash
docker-compose up -d
```

### 3.5 查看服务状态

```bash
docker-compose ps
docker-compose logs -f aiot-backend
```

### 3.6 验证服务

```bash
# 测试后端 API
curl http://localhost:8080/api.json

# 测试 EMQX
curl http://localhost:18083

# 测试前端
curl http://localhost:5173
```

## 四、ESP32 硬件连接云端

### 4.1 修改固件 WiFi 配置

编辑 `firmware/src/config.h`：

```cpp
// ==================== WiFi 配置 ====================
#define WIFI_SSID       "你的WiFi名称"
#define WIFI_PASSWORD   "你的WiFi密码"

// ==================== MQTT 配置 ====================
// ⚠️ 关键：改为你的云服务器公网 IP
#define MQTT_BROKER     "你的公网IP"
#define MQTT_PORT       1883
#define MQTT_CLIENT_ID  "ESP32-CeilingLight-01"
```

### 4.2 修改后端 MQTT 配置

编辑 `aiot/aiot-v2/manifest/config/config.yaml`（本地开发用）：

```yaml
mqtt:
  broker: "tcp://你的公网IP:1883"  # 改为云服务器公网IP
```

Docker 环境用 `config.docker.yaml`（已配置为 `tcp://emqx:1883`，无需修改）。

### 4.3 烧录固件到 ESP32

```bash
# 进入固件目录
cd firmware

# 编译
pio run

# 烧录（首次需按住 BOOT 按键）
pio run -t upload

# 串口监控（查看连接状态）
pio device monitor
```

### 4.4 验证设备上线

1. 打开浏览器访问 `http://你的公网IP:18083`
2. 登录 EMQX Dashboard（用户名: admin，密码: public）
3. 进入「连接」页面，应该能看到 ESP32 设备已连接
4. 打开前端页面 `http://你的公网IP:5173`
5. 登录后在「实时数据」页面应该能看到传感器数据

## 五、数据流转验证

```
ESP32 → WiFi → 云服务器:1883 (EMQX) → Go后端 → MySQL/InfluxDB → 前端展示
                                         ↓
                                    告警检测 → 邮件通知
```

验证步骤：
1. ESP32 串口显示 `MQTT connected` ✓
2. EMQX Dashboard 显示设备连接 ✓
3. 前端「实时数据」页面显示传感器数据 ✓
4. 触发火焰告警（将 IO4 接 GND）→ 收到邮件 ✓

## 六、常见问题

### Q1: ESP32 连不上 MQTT Broker
- 检查云服务器安全组是否开放 1883 端口
- 检查 WiFi 名称和密码是否正确
- 串口查看错误码：`rc=1` 认证失败，`rc=2` 协议错误，`rc=3` 服务器不可达

### Q2: 前端页面打不开
- 检查安全组是否开放 5173 端口
- 检查 Docker 容器是否运行：`docker-compose ps`

### Q3: 传感器数据不上报
- 检查 MQTT Broker 地址是否正确（应为公网 IP）
- 检查 EMQX Dashboard 是否有设备连接
- 串口查看 `[MQTT] Published sensor data` 日志

### Q4: 邮件告警收不到
- 检查 SMTP 配置（QQ 邮箱需开启 SMTP 服务）
- 检查 `config.yaml` 中的邮箱授权码

## 七、生产环境建议

1. **EMQX 认证**：开启用户名密码认证，避免未授权设备连接
2. **HTTPS**：使用 Nginx 反向代理 + SSL 证书
3. **数据库备份**：定时备份 MySQL 和 InfluxDB
4. **监控**：部署 Prometheus + Grafana 监控系统状态
5. **日志**：使用 ELK 或 Loki 收集日志
