# 智能建筑消防逃生系统 — 项目进度与技术文档

## 一、项目简介

一套集 **环境监测 → 火灾预警 → 逃生路径规划 → 智能指示牌控制** 于一体的 AIoT 解决方案。
模拟场景为一栋 3 层商城/办公楼（1F 商铺 + 2F 办公 + 3F 展厅/机房）。

---

## 二、系统架构

```
┌─────────────────┐   MQTT    ┌─────────────┐   HTTP   ┌──────────────┐
│  ESP32 传感器    │ ────────> │  EMQX Broker │ ───────> │  Go 后端      │
│  (DHT22/火焰/   │           │  (MQTT v5)   │         │  (GoFrame v2) │
│   电流/AQI)     │           └─────────────┘         └──────┬───────┘
└─────────────────┘                                          │
┌─────────────────┐   MQTT    ┌─────────────┐                │
│  ESP32 指示牌    │ <──────── │  EMQX Broker │ <─────────────┘
│  (方向箭头/RGB/  │  control  │             │   publish
│   蜂鸣器/LED)   │           └─────────────┘
└─────────────────┘
                                          ┌──────────────┐
                                          │  Python 算法  │
                                          │  (FastAPI)    │
                                          │  蚁群算法 ACO │
                                          └──────────────┘
```

| 后端层 | Go + GoFrame v2 | REST API + MQTT 消费者 + WebSocket 实时推送 |

## 三、已完成事项及关键技术

### 3.1 Go 后端服务 (`aiot-v2/`)

#### 3.1.1 框架与项目结构

- **框架**：GoFrame v2 (`github.com/gogf/gf/v2`)
- **架构**：分层架构 — `api` → `controller` → `logic` → `dao` → `model`
- **路由分组**：
  - 公开路由（无认证）：登录接口
  - 受保护路由（JWT 中间件）：所有管理接口
  - WebSocket 端点：`GET /api/ws/device-data`

#### 3.1.2 管理员认证 (JWT)

- **技术**：GoFrame JWT 中间件 + bcrypt 密码哈希
- **实现位置**：`internal/middleware/auth.go`、`internal/logic/admin/admin.go`
- **功能**：登录 / 注册 / 修改密码 / 获取用户列表 / JWT Token 签发与校验
- **路由**：`/api/auth/login`（公开）、`/api/admin/*`（受保护）

#### 3.1.3 MQTT 消费者

- **技术**：`github.com/eclipse/paho.mqtt.golang` MQTT 客户端
- **实现位置**：`internal/logic/mqtt/mqtt.go`
- **订阅主题**：
  - `sensordata` — 传感器数据上报
  - `signboard/heartbeat` — 指示牌心跳
- **消息格式**：外层 key 为设备 MAC 地址，value 为传感器数据对象
- **处理流程**：
  1. 解析 JSON，按 MAC 区分设备
  2. Redis `SETEX` 刷新设备在线状态（TTL=90s）
  3. 异步写入 InfluxDB v2 时序数据
  4. 异步通过 WebSocket 推送到前端
  5. 异步检测告警条件（火焰/电流异常）

#### 3.1.4 InfluxDB v2 时序存储

- **技术**：`github.com/influxdata/influxdb-client-go/v2`
- **实现位置**：`internal/cmd/cmd.go`（初始化）、`internal/logic/mqtt/mqtt.go`（写入）
- **Measurement**：`sensor_data`
- **Tag**：`mac_address`
- **Fields**：`temp`、`hum`、`AQI`、`TVOC`、`ECO2`、`electricity`、`buzzer`

#### 3.1.5 WebSocket 实时推送

- **技术**：GoFrame 原生 WebSocket Handler
- **实现位置**：`internal/controller/ws/ws.go`、`internal/logic/ws/ws.go`
- **功能**：传感器数据通过 MQTT 接收后，实时推送到已连接的前端客户端
- **端点**：`GET /api/ws/device-data`

#### 3.1.6 设备告警系统

- **实现位置**：`internal/logic/device_alert/device_alert.go`、`internal/logic/mqtt/mqtt.go`
- **告警类型**：
  - 火焰告警（`ESP32_fires_flag=true`）
  - 电流异常告警（`ESP32_electricity_flag=true`）
  - 设备离线告警（Redis TTL 超时）
- **去重逻辑**：`CreateIfNotExists` — 同一 MAC + 同一告警类型只创建一次活跃告警
- **状态流转**：未处理(0) → 已处理(1) / 已忽略(2)

#### 3.1.7 邮件通知

- **技术**：SMTP over SSL（QQ 邮箱 SMTP 服务）
- **实现位置**：`internal/logic/email/email.go`
- **触发时机**：火焰告警 / 电流异常告警创建后自动发送
- **邮件记录**：写入 `notification_log` 表

#### 3.1.8 指示牌管理

- **实现位置**：`internal/logic/signboard/signboard.go`
- **功能**：
  - CRUD 增删改查（含 Redis 缓存）
  - 更新后自动通过 MQTT 发布控制指令（`signboard/control`）
  - 批量查询在线状态（Redis）
- **控制指令格式**：`{ "MAC": { "display_left": "Left", "RGB_left": "#FF0000", ... } }`
- **显示内容映射**：1-10 编号映射到 Left/Right/Up/Down/style/motifs 等

#### 3.1.9 传感器/控制板管理

- **传感器 CRUD**：`internal/logic/sensor/sensor.go`
- **控制板 CRUD**：`internal/logic/board/board.go`
- **传感器-节点绑定**：`internal/dao/sensor_node_binding.go` — 将传感器关联到建筑图节点

#### 3.1.10 逃生路径计算接口

- **实现位置**：`internal/logic/escape/escape.go`
- **功能**：
  1. 从 MySQL 加载 `graph_node` + `graph_edge` 建筑图数据
  2. 火灾模式下过滤电梯节点及其连接边
  3. 加载传感器-节点绑定数据
  4. 组装请求 → 调用 Python 算法服务 (`POST /api/escape/calculate`)
  5. 返回多条逃生路径（按代价排序）

#### 3.1.11 数据报表

- **实现位置**：`internal/logic/report/report.go`
- **接口**：
  - `GET /api/reports/overview` — 概览统计
  - `GET /api/reports/trend` — 趋势数据
  - `GET /api/reports/alarm-stats` — 告警统计
  - `GET /api/reports/export` — CSV 导出

#### 3.1.12 数据库表

| 表名 | 用途 | 关键字段 |
|------|------|----------|
| `admin` | 管理员 | user_account, password(bcrypt), identity(1=管理/2=超管) |
| `board` | ESP32 主控板 | name, status |
| `sensor` | 传感器节点 | mac_address(唯一), address, board_id |

### 3.2 Python 算法服务 (`algorithm-service/`)

#### 3.2.1 蚁群算法 (ACO)

- **技术**：Python + FastAPI + NumPy
- **实现位置**：`algorithms/aco.py`、`main.py`
- **算法原理**：
  1. 构建加权邻接表，结合传感器数据动态计算边权重
  2. 火焰区域权重设为 99999（几乎不可通行）
  3. 高温(>50℃) / 高 AQI(>200) 区域增加危险系数
  4. 蚂蚁在图上行走，基于信息素 + 启发式函数选择下一节点
  5. 迭代 200 轮，每轮 30 只蚂蚁，更新信息素矩阵
  6. 返回到每个出口的最优路径

- **默认参数**：
  - 蚂蚁数量：30
  - 信息素权重 α=1.0，启发式权重 β=3.0
  - 蒸发率 ρ=0.3，信息素常量 Q=100
  - 最大迭代：200
  - 危险系数权重：10.0

- **逃生时间估算**：假设步行速度 1.5 m/s

#### 3.2.2 API 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/api/escape/calculate` | 通用逃生计算入口（按 algorithm 字段分发） |
| `POST` | `/api/escape/aco` | 蚁群算法专用入口 |
| `GET` | `/health` | 健康检查 |

#### 3.2.3 数据模型

- `GraphNode`：节点标识、名称、类型、3D 坐标、楼层、是否出口
- `GraphEdge`：起止节点、距离、基础权重、通道宽度、是否双向
- `SensorReading`：MAC、节点 key、温湿度/AQI/火焰标志、影响半径
- `PathResult`：出口节点、路径节点列表、总代价、预估时间

### 3.4 ESP32 固件 (`firmware/`)

#### 3.4.1 MQTT 通信模块

- **技术**：PubSubClient (`knolleary/PubSubClient@^2.8`)
- **实现位置**：`mqtt_handler.cpp`、`mqtt_handler.h`

- **实现位置**：`hardware_init.cpp`、`hardware_init.h`
- **支持的硬件**：
  - DHT22 温湿度传感器 (GPIO26)
  - 火焰传感器 (GPIO4)
  - 电流检测 ADC (GPIO34)
  - PWM LED 灯带 (GPIO23, P-MOS 驱动)

  - 散热风扇 (GPIO5, P-MOS 驱动)
  - 状态指示灯 (GPIO19)
- **硬件测试**：`hardwareTest()` 函数逐一测试所有外设

#### 3.4.3 配置管理

- **实现位置**：`config.h`
- **内容**：WiFi/MQTT 连接参数、引脚定义、PWM 配置、定时器间隔、告警阈值
